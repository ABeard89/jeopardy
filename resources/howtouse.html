Using the Game Board⇧

  • Select "Start Game" from the Start Screen.
  • On the following screen, choose a Data Set (Game Board) by selecting "Use" within the Data Set.
    • The chosen Data Set will be outlined in green.
    • If no Data Set is chosen, a default one will be used.
    • If there are no Data Sets available, select "Edit Data Sets" from the left pane to go to the Data Set Management Screen.
  • After a Data Set has been chosen, select "Start". The chosen Data Set's Game Board will be displayed.
  • Any cell, other than Category Names, can be selected.
  • After a question cell has been selected, the appropriate question will be displayed, along with:
    • "Cancel" - Goes back to the Game Board without applying points to Teams or closing the question.
    • "Close" - Goes back to the Game Board, applies point changes to Teams and marks the question cell in black.
    • Any configured Teams will display below the "Cancel" and "Close" buttons on the left side.
      • The "X" button signifies an incorrect answer and will deduct the question's value from the Team's score.
      • The "O" button signifies a correct answer and will add the question's value to the Team's score.
  • Questions on the Game Board will be marked
    • with a green outline if it was the previously selected question,
    • with a black background if it has been answered,
    • and/or with a blue background if it has yet to be answered.
  • To return to the Start Screen, refresh the page. (This will reset the Game Board.)

Setup⇧

From the Start Screen, select "Setup". The Options Screen will be displayed. On this screen, any available options can be selected, and custom Teams can be created.

There are several buttons in the left pane of the window:

  • "Back" - Goes back to the previous screen and discards any unsaved changes to Teams or Options.
  • "Save" - Saves the current Teams and Options to local memory.
  • "Data Sets" - Goes to the Data Set Management Screen, where Data Sets containing custom Game Boards are created and managed.
  • "Import" - Goes to the Import Screen where exported data can be imported.
  • "Export" - Goes to the Export Screen where Teams and Data Sets can be exported.
  • "Clear Data" - Clears all currently saved Team and Data Set data. Press with care!

Teams⇧

Teams can be created from the Options Screen by inputting a Team Name and selecting "Create".

Team Points and a Team Color can also be set if desired, but will default to '0' and 'Red' if not specified.

To edit a previously created Team, select "Edit" within the Team.

  • "Edit" will change to "Done", and "Delete" will appear.
  • The Team Name, Team Points, and Team Color will all become editable.
  • Change any field and select "Done".

After any changes have been made,

  • select "Save" in the left pane to save the changes, or
  • select "Back" to discard the changes.

Please note the following cautions:

  • All Team names must be unique. Therefore,
    • a Team cannot be created with another Team's Name,
    • a Team cannot be edited to have another Team's Name.
  • In addition to the previous rules, if a Team's name has been changed, both the new and old name will be unavailable until the next "Save". Therefore,
    • when changing a Team's name, note that another Team cannot have that Team's old name until all changes are saved.

Data Sets⇧

From the Options Screen, select "Data Sets" from the left pane to go to the Data Set Management Screen. From here, Data Sets can be created and edited in the same fashion as Teams.

The same rules apply for Data Sets' names as with Teams' names.

The Game Board for any created Data Set can be customized by selecting "View Board" within the Data Set.

"Back" and "Save" function the same as on the Options Screen. Please note that Game Board changes are also included. If significant changes have been made to a Game Board, it is advisable to save the data as soon as possible.

Customizing the Game Board⇧

After selecting "View Board" on a Data Set, the Make Board Screen will appear. From this screen, the Game Board can be freely customized.

  • Columns (categories) can be added or removed.
  • Rows can be added or removed.
  • Columns and Rows can be rearranged by clicking and dragging the arrow (↔) and dropping it on the desired position.
  • Category names and Row values can be edited.
  • Questions can be edited.

The editor is divided into two main sections:

  • the controls area in the top right of the screen and 
  • the display area in the bottom right of the screen.

To add a Column to the Board:

  • Enter a Category Name in the appropriate field and
  • select "Add Column".

To remove a Column from the Board:

  • Select the column (will be outlined in red) and
  • select "Delete Column" in the top right of the screen.

To edit a Category Name:

  • Select the column (will be outlined in red),
  • select "Edit Column" in the controls area ("Edit Column" will change to "Done" and the Category will become editable),
  • change the Category name (must be unique), and
  • select "Done" in the controls area.

To add a Row to the Board:

  • Enter a Row point value in the appropriate field and
  • select "Add Row".

To remove a Row from the Board:

  • Select "Edit" in the appropriate Row ("Edit" will change to "Done", "Delete" will appear, and the value will become editable) and
  • select "Delete" inside the Row.

To edit a Row's value:

  • Select "Edit" in the appropriate Row ("Edit" will change to "Done", "Delete" will appear, and the value will become editable),
  • change the value as desired (must be unique and contain only numbers), and
  • select "Done" inside the Row.

To edit a Question:

  • Select the appropriate Column (will be highlighted in red and the Rows will display that Category's questions) and
  • select the desired question from the Rows.
  • The following screen will preview the question exactly as it will appear during gameplay.
  • From this screen,
    • choose "Edit" (the question will become editable),
    • edit the question as desired using the advanced text editor,
    • select "Done" to close the editor and preview the question, and
    • select "Close" to return to the Make Board Screen.

Local Storage⇧

When selecting "Save" in the Options Screen or in the Data Set Management Screen, the Team or Data Set data will be saved to local memory.

This data will remain until either

  • the application files are moved or
  • the browser's cache is cleared.

While data will remain here for extended periods of time, it is not advisable to only rely on this. This memory should be considered temporary!

For permanent storage, please Export the data and manually save the resulting text to a file.

Please note that:

  • Each browser keeps a separate cache, so, for example, data made in Internet Explorer will be unavailable to Google Chrome without Exporting and Importing.
  • It is impossible for this application to save a file to a user's computer. This is by design for security reasons. Therefore, manually exporting and importing data is the only way to permanently save the application's data.

Exporting / Importing Data⇧

As previously explained in Local Storage, the only method of permanently saving data is to manually Export it and Import it using the following method.

To view the Export and Import Screens, select either "Export" or "Import" from the left pane of the Options Screen.

To Export all data:

  • From the Export Screen, select "Export Data" in the top right area of the screen,
  • copy the text that appears below the button, and
  • save that data in a file of your choice (e.g. Microsoft Word, Notepad, Word Pad, etc.)

To Import data:

  • Copy the previously exported data text,
  • paste that into the text area below the "Import Data" button on the Import Screen, and
  • select "Import Data".

If all data has been successfully imported, it will be shown on the Options Screen and the Data Set Management Screen.

Please note that:

  • Existing data and imported data will be merged together.
  • If a name conflict occurs between existing data and imported data, the imported data will not overwrite the existing data.
